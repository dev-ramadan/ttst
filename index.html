Authentication & Security Layer

1) bcrypt — تشفير كلمة المرور

  • يتم تشفير كلمة المرور قبل حفظها في قاعدة البيانات.
  • عدم تخزين كلمة المرور كنص عادي Plain Text.
  • استخدام bcrypt.hash() أثناء التسجيل.
  • استخدام bcrypt.compare() أثناء تسجيل الدخول.

2) JWT — Authentication

  • بعد نجاح تسجيل الدخول يتم إنشاء Token للمستخدم.
  • يُرسل الـ Token للعميل ويُستخدم في الطلبات المحمية.
  • يحتوي الـ Token على بيانات أساسية مثل userId و role.
  • يتم التحقق منه في الـ Middleware قبل السماح بالوصول للـ API.

3) Roles — Authorization

  • تحديد صلاحيات كل مستخدم حسب دوره داخل النظام.
  • Admin: إدارة النظام بالكامل.
  • Employee: إدارة العمليات اليومية مثل البيع والغسيل.
  • Customer: الوصول إلى بياناته وخدماته فقط.

bcrypt Example

const bcrypt = require("bcrypt");

const hashedPassword = await bcrypt.hash(password, 10);

const isMatch=await bcrypt.compare(password,user.password);

JWT Example

const jwt = require("jsonwebtoken");

const token = jwt.sign(
  { userId: user.id, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: "7d" }
);

Role-based Access Example

Admin Employee Customer

يتم استخدام الـ role داخل الـ Middleware لتحديد من يمكنه تنفيذ عمليات مثل إضافة سيارة أو حذف موظف أو عرض تقارير النظام.

طبقة الأمان في النظام لا تعتمد فقط على قاعدة البيانات، بل تشمل أيضًا Password Hashing و JWT Authentication و Role-based Authorization لحماية المستخدمين والعمليات الحساسة داخل النظام.
